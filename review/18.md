### ✅ What changed

1. **RAG Service:** `src/services/rag/service.py`
   - `RagService` orchestrates retrieval + generation:
     1. Encode question → search Qdrant for top-K chunks (filtered by group_ids)
     2. Deduplicate by doc_id, look up doc titles from Postgres
     3. Assemble context from chunk texts
     4. Call Ollama with `qa_system_prompt + context + question`
     5. Return answer + citations
   - Best-effort fallback if Ollama fails (returns passages instead)
   - Empty results return a polite "no information found" message

2. **Models:** `src/services/rag/models.py`
   - `QuestionRequest`, `AnswerResponse`, `Citation`

3. **API endpoint** in `src/services/api/main.py`:
   - `POST /qa` → `{question, top_k?}` → `{answer, citations[], model}`
   - Reads `llm.qa_system_prompt` from `system_config`
   - Returns early for users with no groups

4. **Tests:** `tests/integration/test_rag_api.py` (6 tests)
   - answer + citations returned, no groups empty, no chunks found
   - empty question 422, Ollama failure fallback, top_k limits chunks

5. **Docs:** `CHANGELOG.md`, `agents.md` updated.

### ⚠️ Warnings (accepted or deferred)

1. **Feature flag not enforced in endpoint** (`src/services/api/main.py`)
   - `feature.rag_qa` exists in `system_config` but the endpoint does not check it.
   - Consistent with existing features; fix globally in follow-up.

2. **MockEncoder used in production** — Same as the search endpoint. Acceptable for MVP; replace with a real embedding model in Phase 08+.

3. **No streaming response** — The answer is returned in a single JSON response. Streaming would improve perceived latency for long answers. Deferred to Phase 08+.

4. **No conversation history** — Each question is independent. Session/conversation support is a future feature.
