### Post-review fix (applied in follow-up commit)

**`migrations/env.py` — Codex P2:** The original code unconditionally
overwrote `sqlalchemy.url` whenever `POSTGRES_URL` was in the shell
environment. The test-fixture conftest already sets `sqlalchemy.url` directly
on the Alembic `Config` before calling `command.upgrade()`, so a stray env var
would silently redirect migrations to the production database instead of the
isolated SQLite test DB.

Fix: gate the env override on `config.cmd_opts is not None`, which is only
true for CLI invocations (i.e. the Compose `migrate` service) and is `None`
for programmatic callers.
