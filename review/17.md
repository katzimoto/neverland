### ⚠️ Warnings (accepted or deferred)

1. **Feature flag not enforced in endpoints** (`src/services/api/main.py`)
   - `feature.annotations` exists in `system_config` but endpoints do not check it.
   - Consistent with existing features; fix globally in follow-up.

2. **SQLite JSON stored as string** — `_parse_json` helper added in API layer to handle SQLite's TEXT JSON storage. PostgreSQL native JSON would work without this. The helper is harmless for Postgres and essential for SQLite.

3. **Hard delete vs soft delete** — Annotations use hard delete (per spec), unlike comments which use soft delete. This is intentional but worth noting for audit requirements.

4. **`text` field in `AnnotationUpdateRequest` lacks `min_length`** — The update model accepts `str | None` without enforcing minimum length. If a client passes `""`, the database will store an empty string. MVP-acceptable since PATCH partial updates are inherently permissive.
