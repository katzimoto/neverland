### ⚠️ Warnings (should fix)

- `CHANGELOG.md` listed Phase 05b under both **Added** and **Planned**. Fixed in this branch.
- `SlowWorker` class exists and is fully tested, but is not wired into a production entry point (no API endpoint, CLI, or background scheduler invokes it). The spec references a Kafka `documents.enrichment` topic; for the current MVP the worker is tested directly and can be wired into a scheduler or consumer in a later phase.
- `SlowWorker._run` does not guard against processing documents with `status = 'deleted'`. A deleted document could be re-enriched if its `translation_quality` is `pending_high`. Consider adding an early return or status check.
- `POST /documents/{doc_id}/translate` allows translation requests on deleted documents (no `status` check). Consider rejecting requests for non-existent or deleted docs more explicitly.

### 💡 Suggestions (nice-to-have)

- Add a `limit` / `offset` parameter to `GET /admin/enrichment-queue` for large queues.
- The `SlowWorker` failure path leaves `translation_quality = 'pending_high'`. Consider allowing retry by resetting to `fast` after a cooldown, or expose a manual retry mechanism.
- `_maybe_auto_enrich` in `PreviewService` does an unconditional `UPDATE`; adding `AND translation_quality NOT IN ('high', 'pending_high')` to the WHERE clause would make the operation idempotent and safer under concurrency.
